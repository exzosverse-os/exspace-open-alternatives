# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly to help us maintain the security of our software.

### How to Report

Please **do not** open a public issue on GitHub for security vulnerabilities. Instead:

1. **Use GitHub's Private Vulnerability Reporting Feature**: Go to the Security tab of this repository and click "Report a vulnerability" to submit your report privately.
2. **Email**: Contact the maintainers directly with the security vulnerability details.
3. **Include the following information in your report**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if available)

### What to Expect

- We will acknowledge receipt of your report within 48 hours
- We will investigate the vulnerability and determine its severity
- We will work on a fix and coordinate a responsible disclosure timeline
- You will be credited for the discovery (unless you prefer to remain anonymous)

## Security Best Practices

When using this project, please follow these security best practices:

- Keep dependencies updated
- Use environment variables for sensitive configuration
- Follow secure coding practices
- Report any suspicious activity or potential security issues

## Supported Versions

Security updates are provided for the following versions:

- Latest version: Full support
- Previous major version: Security patches only
- Older versions: No longer supported

## Security Tools

This project uses the following security tools:

- **Dependabot**: Automated dependency updates
- **CodeQL**: Code scanning for vulnerabilities
- **Secret Scanning**: Detection of exposed secrets
- **Branch Protection**: Required reviews and status checks

## Contact

For security inquiries, please reach out to the project maintainers through GitHub's private vulnerability reporting feature or via email.

Thank you for helping us keep this project secure!
